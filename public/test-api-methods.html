Testing Different API Access Methods

Based on bundesAPI documentation, let's try different ways to access the API:

Method 1: Fetch ALL Products (No Filter)

GET /api/verzeichnis/Produkt?$skip=0&$take=100

Test if we can get the complete product list and filter client-side.

Method 2: Use VerzeichnisTree First

GET /api/verzeichnis/VerzeichnisTree/4
Then find category GUIDs and try:
GET /api/verzeichnis/Produkt?produktgruppe={GUID}

Fetch the tree structure, find correct GUIDs, then query products.

Method 3: Try OData $filter Syntax

GET /api/verzeichnis/Produkt?$filter=produktartNummer startswith '13.20'

Use OData filtering syntax instead of query parameters.

Method 4: Search by Exact Product Code

GET /api/verzeichnis/Produkt?produktartNummer=13.20.12.1234

Try querying by exact 10-digit product codes.

Method 5: Get Tree Node Children

GET /api/verzeichnis/VerzeichnisTree/4/{categoryGUID}

Get children of a specific category node.
